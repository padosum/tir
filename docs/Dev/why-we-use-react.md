### #리액트 #프론트앤드

# 프론트앤드 라이브러리/프레임워크의 역할
- DOM 관리와 상태 변화 관리를 최소화(DOM 요소의 변화가 반복될수록 비효율성이 높아진다.)

# 그 중에 리액트를 사용하는 이유는?
1. component 단위 작성
- 생산성 극대, 용이한 유지보수
- 컴포넌트 재사용 가능

2. JSX(javascript + xml)
3. Virtual DOM
- virtual DOM 개념을 적용한 프론트앤드 라이브러리/프레임워크 중 성공적인 편이라고 할 수 있다.
